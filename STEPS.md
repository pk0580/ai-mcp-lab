# План разработки NeuronLab

В данном документе описаны этапы реализации проекта от базовой структуры до полноценной многоагентной системы.

## 🔹 Шаг 1 — База (Инфраструктура)
- [x] Настройка окружения Laravel + Docker.
- [x] Конфигурация Redis и PostgreSQL (с расширением pgvector).
- [x] Разработка базового API для взаимодействия с системой:
    - [x] Создание миграций для таблиц `runs` (запуски) и `steps` (шаги).
    - [x] Определение моделей `Run` и `Step` со связями.
    - [x] Реализация эндпоинта для запуска агента (`POST /api/runs`).
    - [x] Реализация эндпоинта для отслеживания статуса и истории (`GET /api/runs/{id}`).
    - [x] Базовая валидация входящих промптов.
- **Результат**: “живое приложение” с готовой инфраструктурой.

## 🔹 Шаг 2 — Базовый агент
- [x] Создание ядра `NeuronAgent`.
- [x] Реализация первого инструмента (например, `search tool`).
- [x] Простейший цикл рассуждений (reasoning loop).
- **Результат**: агент делает несколько шагов.

## 🔹 Шаг 3 — Очереди и асинхронность
- [x] Реализация `StepJob` (обработка шага).
- [x] Перевод reasoning loop в асинхронный режим работы через очереди.
- **Результат**: агент работает через queue.

## 🔹 Шаг 4 — Memory (Память)
- [x] Настройка таблицы embeddings для векторного поиска (pgvector).
- [x] Сохранение истории шагов агента.
- [x] Реализация механизма извлечения контекста (retrieval).
- **Результат**: агент “помнит” свои действия.

## 🔹 Шаг 5 — MCP (Multi-agent)
- [ ] Реализация поддержки нескольких типов агентов.
- [ ] Маршрутизация задач между агентами.
- [ ] Создание `AgentTool` (инструмент для общения агентов).
- **Результат**: агенты взаимодействуют между собой.

## 🔹 Шаг 6 — Reflector
- [ ] Добавление этапа анализа ошибок в цикл агента.
- [ ] Реализация логики повторных попыток (retry logic).
- [ ] Улучшение планов на основе прошлых неудач.
- **Результат**: агент становится “умнее”.

## 🔹 Шаг 7 — Debug & Observability
- [ ] Ведение таблицы логирования `agent_steps`.
- [ ] Создание интерфейса (UI или API) логов для мониторинга «мыслей» агента.
- [ ] Трекинг задержек (latency tracking) на каждом этапе.
- **Результат**: можно дебажить reasoning.

## 🔹 Шаг 8 — Оптимизация
- [ ] Ограничение максимального количества шагов (защита от зацикливания).
- [ ] Кеширование частотных запросов.
- [ ] Уменьшение количества вызовов LLM.
- **Результат**: production-like система.
