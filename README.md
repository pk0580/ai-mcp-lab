# AiMcpLab — Multi-Agent AI System с Reasoning Loop

AiMcpLab — это учебный, но приближенный к production проект, реализующий многоагентную систему ИИ на базе Laravel, MCP и AI SDK.

Ключевая особенность проекта — реализация агента с циклом мышления (reasoning loop), способного:
- планировать действия
- выполнять шаги через инструменты (tools)
- анализировать результат
- корректировать стратегию

## 🎯 Цель проекта

Построить систему, которая имитирует “мышление” ИИ-агента и демонстрирует:
- как устроены современные AI-агенты (аналог Neuron / AutoGPT)
- как строится multi-agent архитектура
- как оптимизировать latency и стоимость LLM
- как дебажить reasoning

## 🧠 Концепция агента (Neuron)

Агент работает не как “вызов модели”, а как цикл принятия решений:
**PLAN → ACT → OBSERVE → REFLECT → (loop)**

Пример поведения:
1. **Пользователь**: "Собери информацию о scaling Laravel и дай рекомендации"
2. **Агент**:
   - **Планирует**: "нужно найти данные"
   - **Делает action**: search
   - **Получает результат**
   - **Анализирует**
   - **Делает следующий шаг** (пока не достигнет цели)

## 🏗 Архитектура системы

### 1. Core
- **NeuronAgent** — главный цикл
- **Planner** — генерация шагов
- **Executor** — выполнение действий
- **Reflector** — анализ ошибок
- **Memory** — хранение контекста

### 2. MCP (оркестратор)
- управляет агентами
- распределяет задачи
- позволяет агентам общаться

### 3. AI SDK
- генерация reasoning (thought/action)
- обработка текста
- принятие решений

### 4. Storage
- **PostgreSQL + pgvector** → память
- **Redis** → очереди + кеш

### 5. Async слой
- **Laravel Queue**
- **Jobs** = шаги агента

### 6. Frontend (Интерфейс)
- **Blade + Tailwind CSS** — легкий интерфейс чата.
- **Server-Sent Events (SSE)** — стриминг процесса рассуждений агента в реальном времени.

### 7. Docker
- **app** — основное веб-приложение (Laravel)
- **worker** — стандартный обработчик очередей
- **redis** — хранилище для очередей и кеша
- **postgres** — основная БД с поддержкой векторов
- **horizon** — панель управления и мониторинга очередей

### 8. Мониторинг и инструменты
В проекте предусмотрены инструменты для отладки и мониторинга:
- **Laravel Horizon** — управление очередями. Доступен по пути `http://localhost:8000/horizon`.
- **Adminer** — веб-интерфейс для управления базой данных PostgreSQL. Доступен по адресу `http://localhost:8080`.
  - Хост: `db`
  - Пользователь: `user`
  - Пароль: `password`
  - БД: `mcp_ai`

## 🧩 Ключевые модули

1. **Agent Engine**: запуск reasoning loop, управление шагами.
2. **Task System**: задачи для агентов, статус выполнения.
3. **Tool System**: инструменты агента (search, db, http, agent-to-agent).
4. **Memory System**: short-term (Redis), long-term (pgvector).
5. **Debug System**: логирование `thought` → `action` → `result`.

## 🚀 Вау-фичи (что выделит проект)

1. 🔥 **Полный reasoning loop**: Не просто чат — агент реально “думает”
2. 🔥 **Streaming Reasoning**: Процесс мышления отображается в реальном времени через SSE
3. 🔥 **Multi-agent взаимодействие**: Агенты могут делегировать задачи, спорить/уточнять и объединять результаты
4. 🔥 **Debugging reasoning**: Можно увидеть THOUGHT → ACTION → RESULT прямо в UI
5. 🔥 **Vector memory**: Агент использует прошлый опыт
6. 🔥 **Асинхронный агент**: Работает через очереди, как в production

## 🚀 Быстрый старт

1. Соберите и запустите контейнеры:
   ```bash
   docker-compose up -d --build
   ```
2. Установите зависимости и выполните миграции:
   ```bash
   docker-compose exec app composer install
   docker-compose exec app php artisan migrate
   ```
3. Откройте приложение в браузере: `http://localhost:8000` (Интерфейс чата)
4. Мониторинг очередей (Horizon): `http://localhost:8000/horizon`
5. Управление БД (Adminer): `http://localhost:8080` (хост: `db`, пользователь: `user`, пароль: `password`)
